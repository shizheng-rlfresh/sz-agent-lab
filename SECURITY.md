# Security Policy

Agentic Systems Lab is a learning repo, but security reports are still welcome.

## Reporting A Vulnerability

Please email Zheng Shi at <shi.zheng.tfls@gmail.com> for issues that should not be disclosed publicly right away.

Use a public GitHub issue for non-sensitive hardening ideas, documentation corrections, and reproducibility problems.

## Scope

In scope:

- unsafe behavior in the local lab code;
- path confinement or tool policy bypasses;
- accidental exposure of secrets or private data in the repository;
- misleading security guidance in the book.

Out of scope:

- attacks that require modifying the local repository before running the examples;
- denial-of-service reports against GitHub Pages static hosting;
- model-provider vulnerabilities outside this repository.

## Expectations

Include the affected file, command, observed behavior, and a minimal reproduction when possible. Do not include live secrets or private third-party data in reports.
